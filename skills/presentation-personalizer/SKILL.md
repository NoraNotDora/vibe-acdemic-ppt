# Presentation Personalization Requirements

## How to Use
- Read this file before every new PPT generation or revision cycle.
- Append new special requirements directly to the “Requirement History” section.
- If requirements conflict, the most recent item has priority; record override notes.

## Active Requirements
- Presentation duration: not specified yet (must be confirmed before script generation)
- Presentation perspective: Motivation -> Method -> Evaluation -> Conclusion
- Chapter check: verify coverage section-by-section and identify missing key content
- Figure strategy: maximize source figure coverage and arrange by figure-content linkage
- Output form: generate a direct speech script, no rehearsal script
- Planning rule: create macro-level structure first, then page-level script
